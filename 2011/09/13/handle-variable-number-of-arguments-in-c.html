Reading time : Less than 8 minutes In many applications I had run into situations where I can’t predetermine the number of arguments while defining a function. Or even, some of you might be wondering how the printf() function works fine any number of arguments if proper format string is provided. Suppose If I have to get the maximum value out of a number of variables, see If don’t know the number of variables to be handled, it will be a mess. We would rather use an array, but that is not a good idea when you really want to reflect the idea of handling multiple values in a function signature. It would be better if we can call the same function as either,

max( 5, 35, 17, 22, 87, 11 );max( 3, 72, 33, 10 );

It can reflect the context of the function call. In the above scenario, the first argument is the number of arguments to be handled in the function. The trailing arguments can be stepped through using a macro va_arg() from stdarg.h. Before starting coding, I would like to introduce a couple of other macros also. va_start( va_list ap, last ); type va_arg( va_list ap, type ); void va_end( va_list ap ); The  va_start() macro initializes ap for subsequent use by va_arg() and va_end(), and it must be called first. va_arg() macro always expands to the next argument in the variable argument list. So if it is called consecutively, it will return two consecutive arguments from the list. The va_end() macro can be used to deference the argument list initialized by va_start(). It can be either a function or a macro. Lets see how can we define a variable argument function in C.

#include#includeint max( int count, … )    {    va_list varList;    va_start( varList, count );    int max = va_arg( varList, int ), val = 0;    for(unsigned int i = 1; i < count; i++ )    {        val = va_arg( varList, int );        if( val > max )            max = val;    }    return max;}int main()    {    printf( “%d”, max( 4, 74, 33, 54, 16 ) );    printf( “%d”, max( 6, 23, 23, 12, 68, 17, 31 ) );    return 0;}
