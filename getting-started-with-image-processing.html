Digital Image

Digital images are fundamentally a finite number of pixels distributed in X and Y axis, and are stored on the disk as a two (or more) dimentional array of integers or more complex datastructers, in rows and columns. Any 2D digital image can be viewed as a matrix of size NxM. Each element in the matrix represents a pixel. A pixel, its definition, and how a pixel is displayed on a screen or how it is printed on a paper, are out of the context of this article. There are a few points to be noted here.

  • The image is grayscale - which means, there are only different shades of grey in the palette of colors a pixel in the image can have.
  • All the colors in the palette is indexed - ie, the shades of grays are indexed from 0 to 9.
  • The index of the color of a particular pixel is the value of the corresponding element in the matrix.
  • The display system, reads this value, and look for the color in the color palette using the said number as the index in the palette.

Now, if you think about it, an absolute minimum palette should have at least two colors in it. Suppose my palette of colors have only black and white. Here is an image, that I can represent using this palette.

Now, for indexing two colors, we need a minimum of one-bit of storage. If the bit is 0 —> the color is black, and if it is 1 —> the color is white. Such images are commonly referred to as monochrome images. Every pixel in a monochrome image is either 1 or 0.

But what if I need more colors to realistically reproduce an image? Basically, I need more bits to represent each pixel. Thats it! Suppose instead of just 1 bit, I use 2 bits to represent each pixel. I can now show 4 different colors. Take a look at my new palette.

00 --> Black
01 --> Dark Gray
10 --> Light Gray
11 --> White

Note : You don't have to limit your palette to shades of gray only. I can use any four colors to build my palette. We will cover such palettes down the line.

The number of bits used to represent a pixel is called the bit depth or color depth in digital image processing. Most images we now find on the web has 8-bit or higher depth. Using just 8 bits, you can display 256 different colors.

Mode 13h

While this is an unrelated topic, I think this is worth mentioning about the non-planar graphics mode in VGA. The VGA hardware included a special graphics mode where each pixel drawn on the screen was represented by 8-bits in the graphics memory. There were 256 indexed predefined colors in the palette, and this mode was enabled when the AL value is 0x13 and AH value is 0x00 and BIOS interrupt 10h is executed. Even before this, BBC Micro had a smaller palette of 8 colors using its 3-bit color graphics adapter. Read more about different color depths here. The default 256-color VGA palette is shown below.

Multi-Channel Images (Color Images)

So far we have discussed only single channel images, where each pixel in the image is represented by a single number. This brings one limitation as the number of shades (or colors basically) we want to have in an image increases. Suppose you have an image with some pixels with colors wide apart in the spectrum of colors. For simplicity, I'm representing such an image in a small matrix as shown below.

This is a very small 3-pixel by 3-pixel image. Assuming all the colors in the image is in between 0 and 4294967295. Meaning, the bit-depth required to represent each pixel in this image is now 32-bits, which is very large. This fundamentally means, the size of the file on disk will be large.
