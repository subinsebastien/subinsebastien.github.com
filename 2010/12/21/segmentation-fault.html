Reading time : Less than 5 minutes I came across developing a graphical user interface for a well known command line tool called identify. I figured out that, at a point in the code, using strcat is necessary. Gcc kept giving me executable code which makes Segmentation Fault and I was unable to find what was wrong. The code looked like,

include #include int main()  {    char* x = “Hello”;    char* y = “Guys”;    strcat( x , y );    printf( “%s” , x );    return 0;}

At first site, there was no room for error in this code. But the point here is that, a statement like

char* x= “Hello”;

will create a constant string which cannot be handled by strcat for concatenation. Because the signature for strcat is

char *strcat( char *dest, const char *src )

So, the destination string should be writeable. That means we have to create a non constant string and which should contain the value Hello. A probable solution I’ve found is using malloc function for allocating some writable memory. The new code which works perfectly for me looks like,

include#include#includeint main()  {    char* x = (char*)malloc(512);    strcpy( x , ”Hello” );    char* y = ”Guys”;    strcat( x , y );    printf( “%s” , x );    return 0;}

So, that was the end of the day for a lowlife programmer… :) See you in the afterlife…Subin Sebastian
