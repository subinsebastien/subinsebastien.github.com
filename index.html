Prim-Server (November 2016) — The original intent of the project is to create a mobile app client for the Primitive OSS project. The app directly upload files to an S3 bucket, and a Lambda function is triggered by it. The function pushes a message to an SQS queue, which will be polled by a worker process running on an EC2 instance. Subsequenctly it takes the file from S3, and passes it onto the primitive program. The final output is sent to the user as an FCM notification.
